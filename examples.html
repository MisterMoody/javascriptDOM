JavaScript and the DOM

Planning is the most essential factor when working with <html> elements in the DOM. A clearly defined objective will help determine a specific approach for writing JavaScript functions that coordinate DOM events with user actions. The most common user actions that will trigger an event occur when a user clicks or mouses-over an element, or manually inputs data using the keyboard. For all intents and purposes, only user events are discussed here: page loads, displaying messages or closing windows may be evaluated in the future.

Common Events

The MouseEvent Interface represents any event that comes about as the result of a user interacting with a web page using a pointing device, and emulated on touch devices. The list below includes important events pertaining to this category:

  • mousedown
  • mouseup
  • mouseover
  • mouseout
  • mousemove
  • click  /   dblclick
  • input

A 'click' event is fired when a user presses then releases the primary mouse button inside of an element. This event takes place after both the mousedown and mouseup events have fired. In contrast, a 'dblclick' event is fired when a user clicks an element twice: quickly and without delay.

A 'mousedown' event is fired when a user presses down on the primary mouse button as a user hovers the mouse over an element. In contrast, a 'mouseup' event is fired when a user releases the primary mouse button.

A 'mouseover' event is fired when a user places the mouse pointer over an element, without clicking it. In contrast, a 'mouseout' event is fired when a user moves the mouse and its cursor outside of an element.

The 'mouseenter' and 'mouseleave' events act similarly to the mouseover and mouseout events with one exception: they are only utilized for complex UI scenarios that do not hierarchy bubbling. Because these events can cause performance issues, it is best to use them like the CSS :hover pseudo-class.

A 'mousemove' event is fired when a user moves the mouse while inside an element.

An 'input' event is fired when the value of <input>, <select> and <textarea> elements have been modified.

With commonly used events defined, we can explore how they can be utilized as programmed events.

MISC/Leftover Examples

DOM Traversal is a means of moving from one part of the DOM to another by selecting an element based on its relationship to another element. To do this, simply traverse to the parentNode using the parentNode Property. Take the elements parentNode and assign it to a variable parent, then use the parent to take an action: in this case, use the parent to remove an element (ie paragraph).
