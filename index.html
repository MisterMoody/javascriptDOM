JavaScript and the DOM

The Document Object Model (DOM) is a programming interface for HTML and anything that is viewable in a web browser. The DOM is a data representation of all HTML loaded into the browser.

An Interface is an Object structured to enforce specific properties on an Object (or Class): Interfaces are designed as Objects, and ALL Objects are Properties of the Global Variable Window Interface, which represents a window containing a DOM Document and all of its Objects. The Document serves as an entry point into the web page's content were Properties and Methods are used to control the structure, style, content and functionality of a Document on the web in a multitude of ways. These entry points are specific to the DOM Tree acting as Nodes that are accessible via Selectors aka <html> Elements. Selection is a way to identify an element so that the browser can find and make it available for us to do something with it using JavaScript. The Selector is appended to the document global variable object which will crawl the current page for the selected element, then returns that element.

Working with the DOM generally requires controlling HTML Elements using JavaScript and this process consists of:
[1] Begin by selecting an <html> Element for a specified function that is (or will be) referenced in the script.
[2] The referenced element is utilized by a selector method and appended to the Document Interface, which is stored in a variable that can be used throughout the script.
[3] End by passing the referenced selector variable to an Event Listener and specify its functionality.

[1a] Target and Select <HTML> Elements

HTML elements can be targeted for selection by their class-name, id or actual tag. The <html> model below offers five kinds of tags, a class and an id that could be targeted for selection.

 <div class="className">
   <h2 id="idName">TITLE</h2>
   <p>Lorem ipsum dolar et inuit a lah zo'men</p>
   <ul>
     <li>Item A</li>
     <li>Item B</li>
     <li>Item C</li>
     <li>Item D</li>
   </ul>
 </div>

[1b] Store Element in a Variable for Reference

After selecting an element, utilize it by creating a variable that incorporates the Document interface with a Method that belongs to that document object: from here, the element is stored and will called upon once that variable is activated.

var varName = document.getElementById( '#id' );
var varName = document.getElementsByTagName( 'li' );
var varName = document.getElementsByClassName( '.class' );
var varName = document.querySelector( '#id OR tag OR .class' );
var varName = document.querySelectorAll( '#id OR tag OR .class' );
var varName = document.querySelectorAll( '[attribute=value]' );
var varName = document.querySelectorAll( 'li:[nth-child(odd)]' );

[2a] Utilize Event Listener

With the selected element stored in a variable, the Event Listener Method is invoked to perform the function that is called whenever the event is triggered.

selector.addEventListener('type', () => {...} );

The syntax appends the selector to the .addEventListener() method, which takes two parameters. The first parameter defines the type of event to listen for. (See 2b)

[2b] Add a Function to Element

There are a plethora of properties and methods that can be utilized to assign a function to an element: simply place inside of an event listener.

.addEventListener('click', () => {  selector.property = 'value';  } );

The second parameter represents the function (or object) that is used to manipulate the element when an event is triggered.

Modify Text

Element.textContent can be used to modify text.

  • selector.textContent = 'This is the main title.'
  • selector.textContent = inputSelector.value;
  • selector.textContent = selector.textContent.toUpperCase();
  • selector.textContent = selector.textContent.toLowerCase();
Modify Attributes

Element.attribute gets or sets an elements attribute. Creating a tooltip is an example. Review the List of Attributes.

  • selector.title = 'Tooltip Content'
Style Elements

Element.style.property can be used to set the CSS styles of an element. Toggling the display of an element is an example.

  • selector.style.backgroundColor = 'red';
  • selector.style.color = 'red';
  • selector.style.color = inputSelector.value;
  • selector.style.display = 'none';
Create Elements

Not only can a function be written to modify an element, it can also be used to create a new element as illustrated below. The first line creates a new <ul> variable to store <ul> elements. Events

Anytime you interact with a webpage, you generate different kinds of events. An Event is something that occurs on a webpage such as a user action (ie clicking on a link) to automated notifications. JavaScript was created to listen for events and respond to them: actually, the browser listens for events while JavaScript is used to respond accordingly.

Events are represented by an Object, based on the Event Interface. There are more than 100 JavaScript event-types defined in official specifications that can be utilized to create an event.

The representation of an Object is a Function, and functions can be passed as a Parameter to an Event Listener.

Delay Execution

The setTimeout Function sets a timer which will execute a function once the timer expires.

  • var varName = window.setTimeout();

The Event Listener

The EventTarget.addEventListener() Method registers a listener on the EventTarget (ie element in Document or Window Interface) it's called on. The first parameter is an event-type: the event to listen for. The second parameter is the JavaScript Function (ie Callback Function aka Event Handler) to be implemented when an event takes place.

Traversing the DOM

DOM Traversal is a means of moving from one part of the DOM to another by selecting an element based on its relationship to another element. To do this, simply traverse to the parentNode using the parentNode Property.

Take the elements parentNode and assign it to a variable parent, then use the parent to take an action: in this case, use the parent to remove an element (ie paragraph).
