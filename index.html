JavaScript and the DOM

The Document Object Model (DOM) is a programming interface for HTML and anything that is viewable in a web browser. The DOM is a data representation of all HTML loaded into the browser.

An Interface is an Object structured to enforce specific properties on an Object (or Class): Interfaces are designed as Objects, and ALL Objects are properties of the global variable Window Interface, which represents a window containing a DOM Document and all of its Objects. The Document Interface serves as an entry point into the web page's content were Properties and Methods are used to control the structure, style, content and functionality of a Document on the web. The Document interface represents any web page loaded in the browser and serves as an entry point into the web page's content, which is the DOM tree. It provides functionality globally to the document, like how to obtain the page's URL and create new elements in the document. This interface describes the common properties and methods for any kind of document and inherits from the Node and EventTarget interfaces.

These entry points are specific to the DOM Tree acting as Nodes that are accessible via Selectors (ie <html> elements) that are appended to the document interface where it is stored in a variable. Targeted elements properly stored in this fashion can then be passed to an .addEventListener() method to perform a specified function whenever an event is triggered.

The Process of Working with the DOM

Selecting and manipulating elements while listening for events is the basic gist of working in the DOM. An example of this is when the browser loads a page, <form> fields start 'listening' for the cursor to hover over them. The field itself can 'listen' to its content, in case a user misspells their email.

Selecting an Element

The first step in this process is quite simple: select an element to be used for manipulation and store it in a variable. Selection is a way to identify an element so that the browser can find and make it available for us to do something with it using JavaScript.

Elements can be selected by id, class name or tag: individually or as a collection. The <html> model below highlights elements that could be targeted for selection:

Selection Methods

There are five methods that can be used to select an element when proceeding to store it in a variable, all of which are structured in similar fashion:
Syntax        const varName = document.method('selector');

The .getElementById( 'id' ); method is use when selecting an #id: this is the most commonly used method for manipulating or getting data from an individual element on your document.

The .getElementsByClassName( 'className' ); and .getElementsByTagName( 'li' ); methods are commonly used to return a collection of all elements in the document with the specified .className or <tag>, respectively, as a NodeList object. The NodeList object represents a collection of nodes that can be accessed by index numbers, starting at 0. You can use the length property of the NodeList object to determine the number of elements, then you can loop through all elements and extract the info you want.

The aforementioned methods will return live DOM elements whereas any changes are reflected in the NodeList object. Sometimes it is best not to return live DOM elements and this can be done using a query method.

The .querySelector( '#id' ); and .querySelectorAll( '.class' ); methods return the first element or all elements in a document, respectively, that matches a specified selector, as a static NodeList object. These methods might appear to have magical powers as much can be done for precise specification of targeting elements. Check out some of their cool features:

document.querySelectorAll( '[attribute=value]' );
document.querySelectorAll( 'li:[nth-child(odd)]' );
document.querySelector( 'ul.odds > li:nth-child(odd)' );

[2a] Utilize Event Listener

With the selected element stored in a variable, the Event Listener Method is invoked to perform the function that is called whenever the event is triggered.

selector.addEventListener('type', () => {...} );

The syntax appends the selector to the .addEventListener() method, which takes two parameters. The first parameter defines the type of event to listen for. (See 2b)

[2b] Add a Function to Element

There are a plethora of properties and methods that can be utilized to assign a function to an element: simply place inside of an event listener.

.addEventListener('click', () => {  selector.property = 'value';  } );

The second parameter represents the function (or object) that is used to manipulate the element when an event is triggered.

Modify Text

Element.textContent can be used to modify text.

  • selector.textContent = 'This is the main title.'
  • selector.textContent = inputSelector.value;
  • selector.textContent = selector.textContent.toUpperCase();
  • selector.textContent = selector.textContent.toLowerCase();
Modify Attributes

Element.attribute gets or sets an elements attribute. Creating a tooltip is an example. Review the List of Attributes.

  • selector.title = 'Tooltip Content'
Style Elements

Element.style.property can be used to set the CSS styles of an element. Toggling the display of an element is an example.

  • selector.style.backgroundColor = 'red';
  • selector.style.color = 'red';
  • selector.style.color = inputSelector.value;
  • selector.style.display = 'none';

Events

Anytime you interact with a webpage, you generate different kinds of events. An Event is something that occurs on a webpage such as a user action (ie clicking on a link) to automated notifications. JavaScript was created to listen for events and respond to them: actually, the browser listens for events while JavaScript is used to respond accordingly.

Events are represented by an Object, based on the Event Interface. There are more than 100 JavaScript event-types defined in official specifications that can be utilized to create an event.

The representation of an Object is a Function, and functions can be passed as a Parameter to an Event Listener.

Delay Execution

The setTimeout Function sets a timer which will execute a function once the timer expires.

  • var varName = window.setTimeout();

The Event Listener

The EventTarget.addEventListener() Method registers a listener on the EventTarget (ie element in Document or Window Interface) it's called on. The first parameter is an event-type: the event to listen for. The second parameter is the JavaScript Function (ie Callback Function aka Event Handler) to be implemented when an event takes place.
